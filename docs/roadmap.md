# HBS2026 Roadmap (Prioritized Tasks & Features for Future Development)

This document collects planned tasks, features, and improvements for the HBS2026 Live-Scoring System, organized by priority.

## High Priority

- [ ] Add ability for manual accepting new admin registration by an already existing admin
- [ ] Add input validation and sanitization(Club names must be unique)
- [ ] Implement data backup and recovery
- [ ] Add comprehensive unit tests

## Medium Priority

### Performance & Security
- [ ] Optimize Firebase queries and caching
- [ ] Add rate limiting for admin actions
- [ ] Optimize service worker for better offline performance
- [ ] Add error tracking and monitoring

### Maintenance & Refactoring
- [ ] Refactor code for better modularity
- [ ] Improve documentation and code comments
- [ ] Migrate to newer JavaScript/ECMAScript standards
- [ ] Clean up unused code and assets

### UI/UX Enhancements
- [ ] Add loading states and skeleton screens
- [ ] Add search/filter functionality for clubs

## Low Priority

### New Features
- [ ] Implement leaderboard animations and effects
- [ ] Add ability to maintain list of shot drinks and list of bar drinks
- [ ] Add ability that clubs can order drinks to reduce efforts and waiting times and the barkeepers
- [ ] Add ability to track drink orders and status for barkeepers

### Integrations
- [ ] Add social media sharing buttons(from scoreboard)

### Additional Testing
- [ ] Add comprehensive integration tests
- [ ] Add comprehensive E2E tests
